HTTP (HyperText Transfer Protocol) - протокол передачи произвольных данных прикладного уровня, который изначально был
    предназначен для передачи только HTML (HyperText Markup Language) документов.
Протокол соответствует клиент-серверной архитектуре, где взаимодействие и клиента и веб-сервера осуществляется
    по стандартной схеме "request-response".
Каждое HTTP-сообщение, независимо от того, следует оно от клиента к серверу(request) или от сервера клиенту(response),
    состоит из трех основных частей:
        1. Стартовая строка - определяет тип сообщения, в которой указываются следующие данные (general):
            a. Request:
                    - метод (GET, POST, PUT, DELETE, etc)
                    - адрес (URL)
                    - версия протокола (HTTP/1.1, HTTP/2)
                HTTP метод - это название операции, которая должна быть выполнена HTTP запросом (request):
                    - GET
                    - POST
                    - PUT
                    - DELETE
                    - PATCH
                    - HEAD
                    - CONNECT
                    - OPTIONS
                    - TRACE
                Адрес (URL - Uniform Resource Locator) состоит из 4 основных частей:
                    https://www.example.com:443/users/1?word=java&sourceid=chrome
                    - протокол - https://
                    - доменное имя или IP адрес (с портом) - www.example.com:443 (порт можно опустить если он стандартный)
                    - адрес ресурса (по указанному доменному имени или IP адресу) - /users/1
                    - список параметров - ?word=java&sourceid=chrome
            b. Response:
                    - версия протокола (HTTP/1.1, HTTP/2)
                    - код состояния (1xx - Informational, 2xx - Success, 3xx - Redirection, 4xx - ClientError, 5xx - ServerError)
                        Код состояния (Status code) - цифровой код ответа сервера, состоящий из 3-х значного числа, первая
                        цифра которого означает класс состояния:
                        1xx - Informational - информация о состоянии процесса передачи;
                        2xx - Success - информация об успешном принятии запроса и его обработке;
                        3xx - Redirection - информация о том, что необходимо выполнить запрос по другому адресу,
                            указанному в заголовке location (header);
                        4xx - ClientError - информация об ошибках со стороны клиента;
                        5xx - ServerError - информация об ошибках на стороне сервера;
                    - текстовое пояснение (OK, Not Found, Internal Server Error)
        2. Заголовок (header) - характеризуют тело сообщения (body) и параметры его передачи в виде "header_name:header_value"
            Другими словами, является метаинформацией HTTP сообщения (названия заголовков не чувствительны к регистру)
        3. Тело сообщения (body) - это непосредственно присылаемые данные HTTP сообщением. Отделяются от заголовка (header)
            пустой строкой. Сами данные могут быть совершенно любые, например:
                                        - HTML страницы
                                        - JSON
                                        - XML
                                        - Видео
                                        - Картинки
                                        - Файлы

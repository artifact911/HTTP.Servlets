OSI - Open System Interconnection model - эталонная модель взаимодействия открытых систем, которая лежит в основе всех стандартов
        компьютерных сетей.
        Модель зарработана международной организацией по стандартизации ISO (International Organization for Standardization), поэтому полное
         название ISO/OSI

Уровни OSI:
    7. Прикладной уровень (application layer) - обеспечивает связь между приложениями. Это единственный уровень, который напрямую
        взаимодействует с данными от пользователей.
        Протоколы прикладного уровня: HTTP, FTP, SMTP, SSH и др;
    6. Уровень представления (presentation layer) - основная задача этго уровня заключается в преобразовании передаваемых данных в взаимно
        согласованные форматы, шифровании, компрессии данных и обратные этим процессам - дешифрование, декомпрессия и т.д.
        Протоколы уровня представления: TLS, SSL;
    5. Сеансовый уровень (session layer) - организует сеансы связи между устройствами, позволяя взаимодействовать длительное время. Другими
        словами, ответственных за открытие и закрытие соединений (сессия - то и есть время между открытием и закрытием соединения мнежду
        устройствами). На практике протокомы сеансового уровня не выделяются. Их работу выполняют протоколы верхних уровней;
    4. Транспортный уровень (transport layer) - разбивает поток данных на сегменты для передачи на сетевой уровень (и наоборот, склеивает
        пакеты из сеиевого уровня в сегменты), добавляя свой заголовок к каждому сегменту (порты приложений), а также выполняет процедуры
        для обеспечения необходимого уровня надежности передачи информации.
        Основные протоколы транспортного уровня: TCP, UDP;
            Порт(TCP, UDP) - это некоторое число в диапазоне от 1 до 65535, которое сопостовляется с конкретным приложением (протокол
            прикладного уровня), при помощи которого осуществляется доступ к сети. Появляется возможность на одной машине запускать
            несколько приложений с доступом в сеть по шаблону IP:port
                                                                        127.0.0.1:8080
                                                                        127.0.0.1:7589
    3. Сетевой уровень (network layer) - главной задачей является разбиение сегментов с предыдущего уровня на ПАКЕТЫ данных (и наоборот,
        ПАКЕТЫ собирать в сегменты для отправки на следующий уровень), а также доставка этих ПАКЕТОВ данных между РАЗНЫМИ СЕТЯМИ.
        Название основного протокола сетевого уровня - IP (Internet Protocol).
        Для решения этой задачи вводится своя система адресации (сетевой адрес IPv4 или IPv6) и реализуется механизм маршрутизации
        (определение пути передачи данных между сетями). Для этого нам нужен МАРШРУТИЗАТОР.
        IP адрес (v4) - это 32-битное число, которое принято записывать в виде 4-х октетов (октет = 1 байт = 8 бит), каждое из которых
        разделено точкой и занимает ровно 8 бит и может принимать значения в диапазоне 0-255:
                                                                192.168.0.1
                                                                172.217.16.4
        IP адрес (v6) - это 128-битное число, которое принято записывать в виде 8-ми 4-х-значных шестнадцатиричных чисел, раздеренных ":"
                                                                2001:0db5:0000:0000:0000:ff00:0042:7879
                                                                2001:0db8::ff00:0042:7879 (сократили нули).
        IPv4 пакет состоит из:
            header - включает в себя разную служебную инфу и занимает 20 байт;
            data - данные в пакете;
        header состоит:
            1. Version - версия (IPv4, IPv6) - 4 бита
            2. IHL (Internet Header Length) - размер заголовка (введен из-за поля Options, которое сейчас не используется) и равен 20 байтам
                - 4 бита
            3. TOS (Type Of Service) - состоит из двух полей: DSCP и ECN - 8 бит
            4. Total Length - полный размер всего IP пакета (максимальный размер пакета 65535 байт) - 16 бит
            5. Identification - идентификатор пакета, который используется в случае его фрагментации на куски (у всех кусков будет то же
                значение) - 16 бит
            6. Flags - используется для контроля за фрагментацией пакетов (где начинается фрагментация пакета, где заканчивается и др.)
                - 3 бит
            7. Fragmentation Offset - говорит на сколько байт нужно выполнить смещение от 0 в случае фрагментации пакета - 13 бит
            8. TTL - время жизни пакета (чтобы не блуждал по сети до бесконечности). Определяет максимальное число узлов, через которое
                пакет может пройти перед тем, как будет уничтожен - 16 бит
            9. Protocol - указывается протокол верхнего уровня, например, в случае с TCP - это 00000110(6), в случае с UDP - 00010001(17)
                 - 8 бит
            10. Header checksum - проверяет только целостность заголовка (протокол IP не умеет проверять целостность данных, это делают
                протоколы верхних уровней) - 16 бит
            11. Source IP Address - IP источника (нужен только принимающей стороне, чтобы знать куда слать ответ) - 32 бита
            12. Destination IP Address - IP адрес назначения (маршрутизаторы смотрят именно на это поле при принятии решения, куда
                направить пакет) - 32 бита
            13. Options - не используется - 32 бита

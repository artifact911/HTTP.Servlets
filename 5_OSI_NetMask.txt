IP адреса недостаточно для определения сети, в которой находится компьютер. Для определения адреса используется
    дополнительное 32-битное число - СЕТЕВАЯ  МАСКА.
            255.255.255.0 (11111111.11111111.11111111.00000000)
            255.255.0.0 (11111111.11111111.00000000.00000000)
            255.255.255.240 (11111111.11111111.11111111.11110000)
    192.168.0.1 - цифры это октеты адреса. Представим в виде битов:
        11000000_10101000_00000000_00000001 - биты адреса
        11111111_11111111_00000000_00000000 - биты маски
    255.255.0.0 - октеты маски
    Там, где заканчиваются биты(единицы) маски, там и заканчивается адрес сети. В маске обязательное условие, что нули
    и единицы не могу чередоваться. Как только проставился первый ноль, дальше идут одни нули. После адреса сети идет
    адрес устройства. Следовательно:
        адрес сети: 192.168
        адрес устройства: 0.1
    Можно встретить такую запись:
        192.168.0.1/16 - 16 - это количество бит адреса сети, остальные это адрес устройства.
Маска используется ТОЛЬКО для компьютера отправителя. В этом случае мы способны понять адрес отправителя и маршрутизатор
    поймет, нужно ли ему передавать эту инфу во вне, либо передать эту инфу внутри сети на адрес другого устройства.

IP адреса должны быть уникальны в пределах всей сети Internet. Распределением таких адресов (они еще называются public)
    занимается IANA (Internet Assigned Numbers Authority) - администрация адресного пространства интернета.
    Проблема в том, что IP адрес v4 занимает 4 байта, следовательно не хватает на всех. Поэтому наряду с public IP
    адресами существуют private IP адреса, которые можно самостоятельно назначать в локальных сетях (в таком случае
    доступ в глобальную сеть Internet осуществляется на основе NAT (Network Address Translation) или Proxy)
    Диапазоны private IP адресов:
            10.0.0.0 - 10.255.255.255
            172.16.0.0 - 172.31.255.255
            192.168.0.0 - 192.168.255.255

Можно посмотреть сколько разных маршрутизаторов мы пройдем для достижения какого-то другого IP указав IP приемника либо
    его доменное имя:
    в cmd пишем:
        - windows: tracert yandex.ru
        - unix: traceroute yandex.ru
